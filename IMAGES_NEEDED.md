# Images Needed for KEXP Blog Post

The blog post `content/post/2025-12-23-kexp-now-playing-display.en.md` requires two images that need to be manually downloaded and placed in the correct directory.

## Required Images

The images should be saved to: `static/post/2025-12-23-kexp-now-playing-display.en_files/`

### Image 1: kexp-display-1.jpg
- **Description**: KEXP Now Playing Display showing current song (main display photo)
- **Source URL**: https://github.com/user-attachments/assets/fede8ad3-1433-46d2-82c7-06079c5f8712
- **Save as**: `static/post/2025-12-23-kexp-now-playing-display.en_files/kexp-display-1.jpg`

### Image 2: kexp-display-2.jpg
- **Description**: KEXP display showing logo
- **Source URL**: https://github.com/user-attachments/assets/1a6c9528-74cf-4a09-93a5-c76b702c470c
- **Save as**: `static/post/2025-12-23-kexp-now-playing-display.en_files/kexp-display-2.jpg`

## Why Images Weren't Downloaded Automatically

The GitHub asset URLs are not accessible from the build environment (blocked domain). The images need to be downloaded manually from a browser or environment with internet access to GitHub's CDN.

## How to Add the Images

1. Open each source URL in a web browser
2. Save the image file with the specified filename
3. Place both images in the `static/post/2025-12-23-kexp-now-playing-display.en_files/` directory
4. Commit and push the changes

Once the images are added, the blog post will display them correctly using Hugo's figure shortcode.
